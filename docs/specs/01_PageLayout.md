# レイアウト
- **[全体]**：垂直（画面全体をカバー）
  - **[上端]**：水平 (z-[60], pointer-events-none)
    - 🏷️ ヘッダー (h-12, pointer-events-auto)
  - **[中央]**：垂直（可変領域）
    - 📇 カードリスト (scroll-padding-top/bottom)
    - **[右下]**：自由（オーバーレイ, z-[70]）
      - ➕ カード追加ボタン
  - **[下端]**：水平 (z-[60], pointer-events-none)
    - 📋 フッター (h-[60px], pointer-events-auto)
  - **[前面]**：自由（モーダル, z-[100]）
    - 🗄️ データベースビューアー
    - 🔍 フィルタダイアログ
    
## 個別仕様

### メインページ
**名称**：メインページ、
**種類**：メインコンテナ、
**用途**：アプリの基底レイアウトの保持、
**形状**：全画面、モダン、フラット、
**寸法**：`h-screen` (100vh)、
**文字**：--、
**色彩**：背景：`slate-50`、
**装飾**：選択色：`purple-100`、
**配置**：垂直配置 (`flex-col`)、
**状態**：クライアントマウント制御（フリッカー防止）、
**挙動**：--、

## ロジック仕様
- **シングルページ構成**: ページ遷移は行わず、ステート管理によって表示コンポーネントを切り替える。
- **モーダル管理**: データベース管理等の重量級ツールは、メイン画面を覆うモーダルダイアログ (`DbViewer`) で提供する。
- **スクロール挙動**: ルート要素 (`main`) の `overflow-hidden` によりブラウザ標準のスクロールを抑制し、`CardList` 内で個別にスクロールを処理することで、ヘッダー・フッターの固定レイアウトを実現する。
- **マウント制御**: `mounted` ステートを用いて、SSRとCSRの不整合（ハイドレーションエラー）や初期ロード時のレイアウト崩れを防ぐ。
