# ヘッダー仕様

## 概要
アプリケーション上部に常に表示されるナビゲーションおよびステータス表示領域。

## 構成要素

### 1. タイトルエリア (左側)
- **アプリタイトル (MyLog)**
  - デザイン: 位置情報アイコンのアクティブ時と同じ deep purple (`#b39ddb`) を基本色とする
  - クリックアクション: アプリ状態のリセット（フィルタ・ソート初期化、リスト再描画、DB再読み込み無し）
- **データベース読み込み進捗表示 (ロゴプログレスバー)**
  - 仕組み: アプリ起動時に紫色のロゴの上に白塗りのロゴを重ね、DB読み込み進捗に合わせて白塗り部を下から上へ消去（clip-path）することで、背後の紫色を露出させる
  - DBにデータがない場合は表示しない
  
- **バージョン番号**
  - アプリケーションの現在バージョンを表示

### 2. ステータスエリア (中央寄り)
- **位置情報アイコン**
  - 現在の位置情報取得ステータスを表示
  - 状態: 取得中 / アクティブ / 非アクティブ / エラー
  - クリックアクション: 位置情報取得のON/OFF切り替え
- **カードカウント**
  - 現在リストに表示されているカードの総数を表示

### 3. アクションエリア (右側)
- **ランダムピックボタン**
  - 機能: 現在のリスト（フィルタ適用後）からランダムにカードを1枚選択し、詳細を展開する
- **DBステータスボタン**
  - 機能: データベースビューワーを開く
  - インジケータ: バックグラウンド処理中やデータ読み込み中はアイコンを紫に変更し、状態を示すテキスト（`Loading`, `Sorting`等）を下部に表示する
  - 補足: データ読み込み時は、1000件読み込むごとにCard/Cellカウントの表示もリアルタイムに更新される


## 動的表示制御
全画面で共通の表示制御が行われる。

- **スクロール連動 (全画面共通)**
  - **下方スクロール時**: 非表示
  - **上方スクロール時**: 表示

- **入力操作連動**
  - **input、textarea等のフォーカス時 (入力中)**: 非表示
  - **フォーカスアウト時 (入力完了時)**: 表示
