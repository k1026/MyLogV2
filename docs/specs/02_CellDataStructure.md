# 2. Cellデータ構造

## 2.1 定義
アプリケーションのデータの基本単位を「Cell（セル）」と呼びます。
データベースのレコードはCellを単位に記録し、Cellのデータメンバがカラムになります。

## 2.2 メンバ定義
Cellクラスには以下のデータメンバが定義されています。

| プロパティ名 | 型 | 説明 |
| --- | --- | --- |
| `id` | string | `[Unix timestamp in ms]-[5-digit random uppercase alphanumeric]`のフォーマットでCellを一意に識別します。 |
| `attribute` | string | 分類 (`Card`, `Time`, `Text`, `Task`) があります。Cellに記録する値や使用するUIが属性によって変化します。|
| `name` | string | セルの名前 |
| `value` | string | セルの値 |
| `geo` | string | セル作成時の位置情報（緯度 経度 高度 の順でスペース区切り）。取得不可時はnull。 |
| `remove` | string | セルの除外判定。除外されている場合はUNIXミリ秒のタイムスタンプを記録、そうでない場合はnull。 |

・セルの生成時間：idの前半の数値をUNIXミリ秒として解釈する。

## 属性ごとの定義と挙動

### 1. Timeセル
- **定義**: 時刻情報を保持するためのセル。
- **attribute**: `Time`
- **name**: デフォルトでは "Time"。
- **value**: UNIXタイムスタンプ（ミリ秒）を文字列として保持。生成時に指定がない場合は現在のタイムスタンプが設定される。
- **挙動**: ログの発生時刻などを明示的に記録するために使用される。

### 2. Textセル
- **定義**: 任意のテキスト情報を保持するためのセル。
- **attribute**: `Text`
- **name**: 任意（空文字可）。
- **value**: 任意の文字列。
- **挙動**: メモや詳細説明など、汎用的なテキストデータとして扱われる。

### 3. Taskセル
- **定義**: タスクの完了状態を管理するためのセル。
- **attribute**: `Task`
- **name**: タスクの内容（任意）。
- **value**: 完了時は完了時刻のUNIXタイムスタンプ（ミリ秒）。未完了時は空文字。
- **挙動**: 値が空文字かどうかによって、タスクの「未完了/完了」を判定する。

### 4. Cardセル
- **定義**: 他のセルをグループ化（パッケージ化）するための特殊なセル。
- **attribute**: `Card`
- **name**: "Card" 固定。
- **value**: 含まれるセルの `id` をスペース区切りで保持する。
- **挙動**: `Cell.addCellId()` や `Cell.removeCellId()` メソッドを通じて、所属するセルのリストを管理する。
