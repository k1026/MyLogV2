---

# 規則
- **【重要】このスキル呼び出し時はユーザーの指示がない限りコード実装や修正を決して行わないこと**。

# Step 

1. **調査範囲の決定**：
- ユーザーからの指示を元に調査する範囲を決定する。
- 実装コードだけでなくテストコードも調査の対象とすること。
- 必要に応じて `docs/specs/*.md` の仕様書やプロジェクトのフォルダ構成、使用パッケージなども参照すること。

2. **調査の実行**：
- 調査範囲に基づき、必要なリソースを調査する。
- 調査時に必要なコマンドを実行してよいが、トークン消費効率を考えコストの低いコマンドを優先すること。(例：`ls` での構造把握や `grep` でのキーワード検索)
- 調査時に実装コードやテスト、ファイル構成などに変更を加えてはならない。

3. **報告**：
- 報告のファイル出力は不要である
- 調査結果をユーザーに**日本語で**報告すること。
- 原因と結果を簡明に記述すること。
- 実装や修正を行う場合はその複雑さや難易度も10段階で併せて報告すること。
- 調査対象ではない問題を発見した場合はそれも報告すること。
