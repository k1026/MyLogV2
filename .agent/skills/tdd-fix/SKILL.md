
## Skill: tdd-fix (Step 2: Red-to-Green)
- **Goal**: `test-results.log` を解析し、テストをパスさせるための最小限の実装を行う。
- **Rules**:
    1. プロジェクトルートの `test-results.log` を読み取り、エラーの原因（失敗したテスト名とエラー内容）を特定する。
    2. `any` を一切使わず、Strictな型を維持したまま、テストをパスさせるためのコードを実装ファイルに記述する。
    3. 修正が終わったら「実装を修正しました。再度 npm run test:ai で確認してください」と告げ、必ず停止すること。
