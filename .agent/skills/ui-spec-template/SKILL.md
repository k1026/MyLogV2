# フォーマットルール
- UIの仕様記述はレイアウトを基に大項目を作成し、その中に各デザインを記載する
- 大項目名はレイアウトの表記名で記載すること(ここには配置は記載しなくて良い)
- 項目番号は必要な場合に限り使用してよいが基本的には不要
- ページ内の構造は以下のようにする

「# レイアウト」
「## 共通仕様」
「## (レイアウトで定義した要素名で個別仕様を記述)」

- ロジックの仕様などは上記のUI仕様の構造の後に記載


# レイアウト
- レイアウトは階層構造で表記する
- 階層構造で入れ子に表記してよい
- 位置表示は太文字で表記し必要に応じて種類を増やしてよいが漢字二文字で統一
- 配置方法は位置情報の隣に「：」を挟んで表記する
- 視覚記号で明治して良い
- 以下はサンプル

- **[全体]**：水平
  - **[左側]**：垂直
    - 🖼️ アプリロゴ
    - 🔢 バージョン番号
  - **[中央]**：垂直
    - 📍 位置情報設定
    - 📊 カード統計
  - **[右側]**：水平
    - 🎲 ランダム抽出
    - 🗄️ データベースツール


# デザイン
- 共通仕様と個別仕様の両方に下記のテンプレートを適用する。
- 指定のない項目も空欄で列挙すること。
- 必要に応じて項目を追加して良い(項目名は漢字2字にし既存の項目名を優先して利用すること)。

**名称**：UIの呼称、
**種類**：UIの種類、
**用途**：目的、機能、
**形状**：形状、質感、角、枠線、
**寸法**：サイズ、長さ、高さ、幅、
**文字**：フォント、サイズ、太さ、行間、整列、
**色彩**：色全般、透明度、
**装飾**：エフェクト、シャドウ、グラデーション、
**配置**：位置、スペース、間隔、階層、関係、
**状態**：変化、遷移、
**挙動**：変形、移動、インタラクティブ、
