# 目的
プロジェクトを簡潔に保つ事で開発効率を向上させること。

# ワークフロー
1. プロジェクトを精査し以下に該当するものを調査する
・`.agent\skills\coding-guideline\SKILL.md`に違反するもの
・フォルダ構成やファイル配置の妥当性
・使用していないファイル、フォルダ、リソース、一時ファイルなど
・使用していないパッケージ、ライブラリ
・不要になったビルド成果物やログ、キャッシュ
・コードの重複、コメントアウトによる古い記述、
・使用していない実装コード、テストコード、デザインコード、

2. 調査結果を報告し対応方針を提案し、修正難易度と影響範囲も提示すること。現状のプロジェクトの状態の評価を点数にして併せて提示すること。**【最重要】ユーザーの承認を得るまで修正や変更は決して行わないこと。**
