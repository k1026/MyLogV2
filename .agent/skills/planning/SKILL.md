**【重要】このスキルではコード実装は行わないこと**。

# Step 1：提案

1. **仕様確認**：ユーザーからの指示を元に `docs/specs/*.md`の 仕様書を分析し矛盾点や不整合、不備などが有れば報告する。

2. **要件定義**：実装の要件を定義し、`docs/specs/*.md`の仕様書の作成または更新する。

3. **実装計画**：要件と既存コードの整合性を確認し、実装計画を立てユーザーに提示する。実装計画にはテスト駆動開発を組み込み、AIエージェントによる実行を前提として精度を維持できるよう実装ステップを十分に分解すること。

4. **記録**：ユーザーに要件定義と実装計画を提示し、`docs/plan.md`の内容を新しい要件定義と実装計画、テスト計画で書き換える。
