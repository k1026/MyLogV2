# 規則
- **【重要】このスキル呼び出し時はたとえユーザーの指示があってもコード実装を決して行わないこと**。
- `agent/skills/lightSAT-DD/SKILL.md`での実装を意識すること

# Step 1: 提案

1. **仕様確認**：ユーザーからの指示を元に `docs/specs/*.md`の 仕様書を分析し矛盾点や不整合、不備などが有れば報告する。

2. **要件定義**：実装の要件を定義し、`docs/specs/*.md`の仕様書の作成または更新する。

3. **実装計画**：要件と既存コードの整合性を確認し、実装計画を立てユーザーに提示する。実装計画にはテスト駆動開発を組み込み、Gemini 3 Flash でも精度を維持できるよう実装ステップを十分に分解すること。

4. **記録**：ユーザーに要件定義と実装計画を提示し、`docs/plan.md`の内容を一旦削除した上で新しい要件定義と実装計画、テスト計画を`docs/plan.md`に出力する。
