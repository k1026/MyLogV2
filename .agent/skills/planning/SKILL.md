# 実行ステップ
各ステップはユーザーの承認を得るまで繰り返すこと。

1. **仕様確認**：ユーザーからの指示を元に `docs/specs/*.md`の 仕様書を分析し矛盾点や不整合、不備などが有れば報告する。

2. **要件定義**：実装の要件を定義し、`docs/specs/*.md`の仕様書を作成または更新しユーザーの承認を得る。

3. **実装計画**：要件と既存コードの整合性を確認し、実装計画を立てユーザーの承認を得る。

4. **記録**：承認された要件定義と実装計画を`docs/plan.md`に記録する。
