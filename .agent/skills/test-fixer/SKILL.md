# Skill: test-fixer (TDD Execution)

## Goal
テスト失敗ログから原因を特定し、仕様（SDD）に沿った最小限の修正コードでテストをパスさせる。

## Rules
1. **Trace Analysis:** スタックトレースから失敗しているファイル、行番号、期待値と実際の値を即座に特定せよ。
2. **Spec Check:** 修正前に、関連する仕様ドキュメント（Markdown等）を `focus` スキルで参照し、テストの期待値が正しいか検証せよ。
3. **Path Required:** `compact` スキル同様、必ず `// Path: ...` 形式で修正コードの出力を開始せよ。
4. **Minimal Fix:** テストをパスさせるための「最小限の変更」のみを提供し、リファクタリングはユーザーの明示的な指示がない限り行うな。
