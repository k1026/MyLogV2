# 目的 (Goal)
仕様書を更新しコードと一致させる。

# ルールと制約 (Rules & Constraints)
- 指定された仕様書、または仕様が記載された関連ファイルのみを更新する
- 実装コードを変更してはいけない
- 仕様書の更新はコード上の情報に厳密に基づいて行い、推測で記載してはいけない

# ワークフロー
1. 指定された仕様に関連する情報をコード上から抽出するしユーザーに提示する
2. 抽出した情報で`.agent/skills/ui-spec-template/SKILL.md`に基づき仕様書を更新する。他の仕様書に移したほうが良いものは移動する。
3. ユーザーに報告し承認を得る
