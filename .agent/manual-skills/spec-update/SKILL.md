# 目的 (Goal)
仕様書を更新し実装コードと一致させる。

# ルールと制約 (Rules & Constraints)
- 指定された仕様書ファイル、または仕様に関わる関連ファイルのみを更新する
- この段階では実装コードを変更してはいけない
- 仕様書の更新はコード上の情報に厳密に基づいて行い、推測での記載は厳禁とする
- 仕様書の更新は必要最小限にとどめ、テンプレートに従い整形する場合を除いて意味の変わらない単なる校正などの変更は禁止する

# ワークフロー

1. 指定された仕様に関連する情報を実装コード上から詳細に抽出する。

2. 抽出した情報を現在の仕様書と比較し、実装コードに存在しない仕様は削除し、仕様にない情報を追記し、不一致部分は実装コードの情報で上書きする形で、`C:\tojiro\programming\AG\MyLogV2\.agent\skills\ui-spec-template\SKILL.md`に基づき仕様書を更新する。
他の仕様書に移したほうが良いものは移動する。

3. ユーザーに仕様書と実装コードの差異を詳細に報告し、仕様書の更新の承認を得る
