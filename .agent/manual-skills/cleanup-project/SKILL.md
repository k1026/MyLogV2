# 目的
プロジェクトを簡潔に保つ事で開発効率を向上させること。

# ワークフロー
1. プロジェクトを精査し以下に該当するものを調査する
・使用していないファイル、フォルダ、リソース、一時ファイルなど
・使用していないパッケージ、ライブラリ
・不要になったビルド成果物やログ、キャッシュ
・コードの重複、コメントアウトによる古い記述、
・使用していない実装コード、テストコード、デザインコード、
・`.agent/skills/guideline/SKILL.md`に違反するもの

2. 調査結果を報告し対応方針を提案。現状のプロジェクトの状態の評価も点数にして併せて提示すること。**【最重要】ユーザーの承認があるまで修正などは決して行わないこと。**
